# TweetMark — Twitter 收藏增强插件 需求文档

> 版本:v1.0 草稿 | 日期:2026-07-06 | 状态:待确认

## 1. 背景与目标

Twitter/X 原生 Bookmark 功能太弱:没有备注、没有分类,文件夹功能要付费(X Premium)。收藏的推文越攒越多,变成一个再也不会打开的黑洞。

**目标**:做一个 Chrome 浏览器插件,在用户点击推文原生 Bookmark 按钮的瞬间,自动捕获推文内容、按关键词规则自动归入文件夹,并弹出一个轻量小窗让用户补充「收藏原因」——不打断刷推节奏,但让每条收藏都有据可查、有处可寻。

**一句话定位**:免费版的 Bookmark 文件夹 + 每条收藏都带"为什么存它"。

## 2. 核心用户流程

```
刷推 → 点击推文下方原生 Bookmark 按钮
     → 插件检测到收藏动作,抓取推文内容
     → 关键词规则自动匹配文件夹(AI / 课程 / 股票 / 其他)
     → 推文旁弹出小窗:显示匹配到的文件夹 + 备注输入框
         ├─ 改文件夹 / 填备注 → 回车或点保存
         ├─ 按 Esc 或 5 秒不操作 → 按自动分类默默保存
         └─ 点「不保存」→ 本条不记录(只保留 Twitter 原生收藏)
     → 之后随时点插件图标 → 打开管理页,按文件夹浏览/搜索/编辑/导出
```

## 3. 功能需求

### 3.1 收藏捕获(content script)

- 监听 twitter.com / x.com 页面上推文 Bookmark 按钮的点击(时间线、推文详情页、搜索结果页都要覆盖)。
- **只在"加入收藏"时触发**,取消收藏不弹窗(v1 不做取消同步,见 §7)。
- 捕获字段(基础快照,自动抓取,无需用户操作):
  - 推文正文(完整文字)
  - 作者显示名 + @handle + 头像 URL
  - 推文链接(状态页 URL,作为唯一 ID 去重)
  - 图片/视频封面的 URL(只存链接,不下载文件)
  - 推文发布时间、收藏时间
- 同一条推文重复收藏时,打开已有记录供编辑,不产生重复条目。

### 3.2 信息补充弹窗

- 收藏后在推文附近(或屏幕右下角)弹出紧凑小卡片,包含:
  - 自动匹配到的**文件夹**(下拉可改,可在弹窗内直接新建文件夹)
  - **收藏原因/备注**输入框(单行起步,可扩展多行),自动聚焦
  - 「保存」按钮(回车等效)、「不保存」按钮
- **不打断原则**:
  - Esc 或 **5 秒无操作** → 按自动分类静默保存,弹窗淡出
  - 弹窗不遮挡推文本身,不阻止页面滚动
- 弹窗风格:深色,与 Twitter 界面协调,小而安静。

### 3.3 关键词规则自动分类

- 初始文件夹与规则(设置页可增删改):

  | 文件夹 | 初始关键词(命中任一即归入,不区分大小写) |
  |---|---|
  | AI | AI, GPT, Claude, LLM, 大模型, prompt, agent, 机器学习, ChatGPT, OpenAI, Anthropic |
  | 课程 | 课程, 教程, tutorial, course, 学习, how to, 入门, guide, 干货 |
  | 股票 | 股票, 美股, $ 开头的 ticker(如 $TSLA), stock, 财报, earnings, 仓位, 投资 |
  | 其他 | (兜底,无规则命中时归入) |

- 匹配范围:推文正文 + 作者名。多个文件夹同时命中时,取**规则列表中靠前**的;弹窗里用户可改。
- 规则维护:设置页里每个文件夹一行关键词(逗号分隔),支持文件夹增删、改名、排序(排序即优先级)。

### 3.4 管理页(插件核心界面)

点插件图标打开独立完整页面(chrome extension page):

- **左侧**:文件夹列表 + 每个文件夹的条数;支持新建/改名/删除文件夹(删除时里面的推文移入「其他」)。
- **右侧**:当前文件夹的推文卡片流,每张卡片显示:作者、正文、备注(高亮显示)、收藏时间、原推链接(点击新标签打开)。
- **搜索**:一个搜索框,同时搜正文、作者、备注。
- **编辑**:卡片上可直接改备注、移动文件夹、删除记录。
- **导出**:一键导出全部数据为 JSON(备份/迁移用);二期再考虑 CSV/Markdown。
- **导入**:能导入之前导出的 JSON(换电脑用)。

### 3.5 设置页

- 文件夹与关键词规则管理(见 §3.3)。
- 弹窗自动保存倒计时时长(默认 5 秒,可调/可关)。
- 开关:是否启用弹窗(关掉则全部静默自动分类)。

## 4. 数据模型与存储

- 存储:`chrome.storage.local`(本地,无账号、无服务器、零成本)。
- 数据结构(示意):

```js
{
  folders: [
    { id, name, keywords: ["AI", "GPT", ...], order }
  ],
  tweets: [
    {
      id,            // 推文状态页 URL 提取的 tweet id,用于去重
      url, text, authorName, authorHandle, authorAvatar,
      mediaUrls: [], // 图片/视频封面链接
      tweetTime, savedTime,
      folderId, note
    }
  ],
  settings: { autoSaveDelay: 5, popupEnabled: true }
}
```

- 容量:`storage.local` 默认 10MB(可申请 unlimitedStorage 权限),纯文字+链接足够存数万条。
- 数据安全:换浏览器/重装靠 JSON 导出导入;管理页显眼位置放导出入口。

## 5. 技术方案要点

- **Manifest V3** Chrome 插件;host permissions 仅限 `twitter.com` / `x.com`。
- **收藏事件检测**:content script 用事件委托监听 Bookmark 按钮点击(`data-testid="bookmark"` / `"removeBookmark"` 区分加入与取消),再从按钮向上找到所在 `article` 节点解析推文内容。
- **已知风险**:Twitter 前端是 React SPA 且 DOM 结构/testid 可能改版 → 选择器集中放在一个配置对象里,方便改版后快速修;解析失败时降级为只存 URL + 弹窗提示。
- **不用 Twitter API**(收费且限制多),全部基于页面 DOM,插件本身完全免费运行。
- 管理页/弹窗:原生 HTML/CSS/JS,不引框架,保持单文件级别的轻量(符合本项目 vibe coding 惯例)。

## 6. 明确不做(v1 范围外)

- ❌ 存量收藏导入(二期:开 Bookmarks 页滚动抓取)
- ❌ AI 自动分类(二期可加:接 Claude API 做智能建议)
- ❌ 多设备云同步 / Notion 同步
- ❌ 取消收藏时的双向同步删除
- ❌ 多标签系统(v1 只有单文件夹 + 备注)
- ❌ Firefox / Safari 支持(先 Chrome)

## 7. 二期候选(按优先级)

1. 存量 Bookmarks 导入模式
2. AI 智能分类建议(替代/补充关键词规则)
3. 标签系统(一条推文多个标签)
4. 「稍后细读」待办状态 + 未读提醒
5. 导出 Markdown / CSV

## 8. 验收标准(v1 完成的定义)

- [ ] 在时间线、详情页、搜索页点 Bookmark,弹窗都能正确出现且内容抓取完整
- [ ] 含「GPT」的推文自动落入 AI 文件夹;含「$TSLA」落入股票
- [ ] Esc / 5 秒不操作会静默保存;填了备注回车保存,备注在管理页可见
- [ ] 管理页可搜索、改备注、移动文件夹、删除
- [ ] 导出 JSON 后清空数据再导入,数据完整还原
- [ ] Twitter 改版导致解析失败时,不报错崩溃,至少存下 URL
